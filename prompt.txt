Build a premium, high-performance 3D interactive portfolio using React, Three.js (React Three Fiber), and GSAP for scroll-driven animations.  ### 1. Concept & Visual Identity - **Theme**: A "Futuristic Neon Journey." The user should feel like they are driving a car through a high-tech digital landscape. - **Aesthetics**: Use a dark mode palette with vibrant neon accents (Cyan, Magenta, Electric Blue). Implement a "Synthwave" or "Cyberpunk" look with glowing grids, distant low-poly mountains, and a dynamic star field. - **Camera**: A third-person follow-camera attached to a car, with smooth damping and slight "jiggle" for realism.  ### 2. Core Mechanics (The Journey) - **Scroll-Linked Animation**: Use GSAP ScrollTrigger to move a 3D car model along a predefined splined path as the user scrolls.  - **The World**: Create a procedural road or infinite grid system. As the car moves, the environment should transition through different "biomes" (e.g., Neon City for About, Desert Grid for Experience, Cyber-Tunnels for Contact). - **Interactive Milestones**: Place 3D "nodes" or floating holographic displays along the road that represent Portfolio sections (Projects, Skills, Experience). When the car reaches a node, the car slows down or rotates to highlight the content.  ### 3. Feature Set - **Dynamic HUD**: A 2D overlay (HTML elements via Tailwind) that shows "Speed" mapped to scroll velocity and "Distance Traveled" mapped to scroll progress. - **Project Holograms**: Interactive 3D cards that use glassmorphism effects. When hovered, they should scale or glow brighter, showing project details and GitHub links. - **Skill Ball Field**: A section where the car drives through a field of floating 3D spheres (using Decals for tech logos) that react to the car's proximity. - **Night-to-Day Transition**: The background lighting should shift from deep night (for Hero) to a sunset/sunrise glow as the user reaches the end.  ### 4. Technical Requirements - **Framework**: React with Vite. - **3D Engine**: @react-three/fiber and @react-three/drei for GLTF loading (Car model) and environment handling. - **Optimization**: Use `Instances` for the grid/stars and `Preload` for models to ensure 60FPS performance. - **Contact Form**: An integrated 3D console or dashboard inside the car scene where users can type messages (powered by EmailJS).
